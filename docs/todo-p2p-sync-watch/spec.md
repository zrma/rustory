# p2p-sync Watch Mode

## 문제
현재 `rr p2p-sync`는 1회 실행 후 종료한다.
실사용에서는 주기적으로 동기화를 돌리고 싶고, 이를 외부(cron/launchd)에만 맡기면 설정/운영 비용이 늘어난다.

## 목표
- `rr p2p-sync`에 watch 모드를 추가한다.
  - `--watch`: 종료하지 않고 주기적으로 동기화 반복
  - `--interval-sec <n>`: 반복 주기(초)

## 동작
- watch 모드에서는 다음을 반복한다.
  1) 한 번의 sync를 시도한다.
  2) 성공/실패는 로그로 남긴다.
  3) `interval-sec`만큼 대기 후 다음 루프로 간다.
- 일부 실패/일시 실패는 프로세스를 종료시키지 않고 다음 루프로 진행한다.

## 범위
- `src/cli.rs`: `p2p-sync` 옵션 추가 및 watch 루프

## 비목표
- 백그라운드 데몬화/자동 시작
- 더 복잡한 스케줄 정책(크론식, 지터, 동적 백오프)

## 결정 사항
- 기본 주기: 60초
